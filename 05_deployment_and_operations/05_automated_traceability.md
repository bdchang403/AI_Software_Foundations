# Module 5: Automated Traceability & The AI Software Bill of Materials (AI-SBOM)

## 🧠 Theory: You are the Auditor of the Machine

In the Context Factory, your role is to ensure **Automated Traceability**. This means that no code enters your production environment unless it has a verified "Chain of Custody" from a human strategic intent.

If a builder uses a "Shadow AI" (an unapproved personal tool) to write code, they are bypassing your security and architectural standards. Your job is to build the **"Audit Gate"** that detects and blocks this.

---

## 🏗️ 1. The AI-SBOM (Software Bill of Materials)

Traditional SBOMs track libraries (e.g., "We use React v18"). An **AI-SBOM** tracks **Cognitive Provenance** (e.g., "This specific function was generated by GPT-4o using EPIC-101 as context").

### Why this is a BuilderOps Requirement:
If a model is later found to have a security "backdoor" or a bias towards inefficient algorithms, the AI-SBOM allows you to run a single command to find and audit every affected file. Without it, you are flying blind.
- **The Input:** Was the `STORY-ID` provided?
- **The Engine:** Which model (e.g., GPT-4o, Gemini 1.5 Pro) generated this code?
- **The Context:** Does the code commit include the mandatory **AI Trailers** (`AI-Generated-By`)?

**Influence:** By enforcing these trailers, you create a "Context Map." You can now tell exactly which files were written by humans and which were generated by specific AI agents.

---

## 🔍 2. Literal Code: The Audit Gate (GitHub Action)

Your CI/CD pipeline acts as a **Mathematical Validator**. 

### 📜 `traceability_check.yml` (The Pattern)
```yaml
name: AI Traceability Audit
on: [pull_request]

jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          
      - name: Verify AI Trailers
        run: |
          # Use grep to find the mandatory trailer in the last commit
          if ! git log -1 --pretty=%B | grep -q "AI-Generated-By:"; then
            echo "❌ ERROR: No AI Context Trailer found. PR Rejected."
            exit 1
          fi
          echo "✅ AI Context Verified."
```

## 🛠️ Exercise: Configuring the Audit Gate

**Scenario:** You need to block any PR that uses an unapproved model (e.g., a non-enterprise local LLM).

**Your Action:**
1.  **Define the Rule:** Modify the `grep` command above to look for specific approved engines (e.g., `gemini-1.5-pro` or `gpt-4o`).
2.  **Enforce the Baseline:** If the engine is anything else, the script must fail.
