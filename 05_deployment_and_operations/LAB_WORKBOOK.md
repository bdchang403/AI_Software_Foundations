As a BuilderOps Engineer, you govern the end of the Context Factory assembly line. In this lab, you will build the guardrails that prevent AI-generated infrastructure from corrupting your cloud state.

## Lab 1: Building the Infrastructure Registry (Module 4)

**The Scenario:**
A builder is asking an AI to build a new Lambda function. You need to ensure the AI uses the existing `AppRole` and `VPC` rather than creating new ones.

### Your Task
Create a file named `infra_registry.json` in your repository that provides the AI with the following "Immutable Map":
- **VPC ID:** `vpc-12345`
- **Private Subnets:** `subnet-abc`, `subnet-def`
- **IAM Role ARN:** `arn:aws:iam::1234567890:role/AppExecutionRole`

**Answer Key (Compare Your Work):**
```json
{
  "context_version": "1.0",
  "approved_infrastructure": {
    "vpc_id": "vpc-12345",
    "subnets": ["subnet-abc", "subnet-def"],
    "execution_role": "arn:aws:iam::1234567890:role/AppExecutionRole"
  }
}
```

## Lab 2: The Traceability Audit Gate (Module 5)

**The Scenario:**
You must block any PR that was generated by an unapproved AI tool.

### Your Task
Write a bash script snippet that would run in a GitHub Action. It must:
1.  Read the last commit message.
2.  Check for the trailer `AI-Generated-By: Enterprise-Agent`.
3.  Fail if the trailer is missing.

**Answer Key (Compare Your Work):**
```bash
#!/bin/bash
TRAILER=$(git log -1 --pretty=%B | grep "AI-Generated-By:")

if [[ "$TRAILER" == *"Enterprise-Agent"* ]]; then
  echo "✅ Traceability Verified: Code generated by approved enterprise tool."
  exit 0
else
  echo "❌ REJECTED: Code contains unverified AI contributions or missing metadata."
  exit 1
fi
```

## Lab 3: AI-SBOM Traceability

**The Scenario:**
An auditor asks, "Which model was used to generate the encryption logic in PR #42?"

### Your Task
If your pipeline is working correctly, where would you find this information?

**Answer Key:**
1.  **PR Description:** The `Blueprint-ID` maps to the Strategy Lead's Epic or the Requirements Lead's Story.
2.  **Git Commit Trailers:** The `AI-Generated-By` trailer includes the model name (e.g., `gemini-1.5-pro`).
3.  **Audit Logs:** Your CI/CD log for the "Traceability Check" records the validation timestamp.
