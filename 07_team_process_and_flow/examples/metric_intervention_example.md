# Example: Telemetry Intervention

Agile Leaders must read the telemetry generated by the BuilderOps Pipeline natively.

## ❌ The BAD Example: Ignoring the Logs
**Scenario:** A builder team completes a sprint 50% faster than usual using AI tools.
**Process Coach Reaction:** *Claps.* "Great job team, the Context Factory is working. Let's do even more points next sprint."

### Why this fails the enterprise:
Faster is not better if it is unconstrained. If the Coach isn't reading the telemetry, they don't see that the team accomplished this speed because the QA automation pipeline was bypassed, and 0% of the AI-generated code was test coverage. Technical debt just exploded.

---

## ✅ The GOOD Example: The Dashboard Audit
**Scenario:** A builder team completes a sprint 50% faster than usual.

1. **The Audit:** The Process Coach pulls the dashboard powered by `github_action_trace_example.yml`.
2. **The Metric:** `Lead Time: 20 minutes (Amazing). PR Size Ratio: 50% Feature / 50% Test (Perfect).`
3. **The Red Flag:** They check the `Constitutional AI <CRITIQUE>` logs exported from the Agent trace.
   - *Log entry:* "Attempt 1: Rejected. Violation of Article III - Used 3rd party library `moment.js` instead of internal `<DateUtil>`."
   - *Log entry:* "Attempt 2: Rejected. Violation of Article III - Used `axios` instead of internal `<HttpClient>`."
   - *Log entry:* "Attempt 7: Success. Used `<HttpClient>`."

**The Intervention:**
The Process Coach walks over to the Builder.
*"Your agent took 7 loops to figure out our HTTP framework. That's a waste of tokens. Why didn't you provide the `Function Registry` schema for HttpClient in your System Prompt Chain?"*

### Why this succeeds:
The Coach identified exactly where the factory assembly line was dropping the context, intervened, and saved the company thousands of API tokens and future compute cycles.
