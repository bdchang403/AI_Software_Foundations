# Capstone Lab: Assembling the Autonomous Agent

In this lab, you will act as the **Orchestrator**. You will take the artifacts generated by the Strategy Lead, Requirements Lead, Builder, and Validation Lead to build a final `.github/agents/` configuration.

## Prerequisites
Before starting this lab, ensure you have the following artifacts ready (or use the examples in the repo):
1.  **Semantic Blueprint** (from Strategy Lead Lab)
2.  **Context-Enriched User Story** (from Requirements Lead Lab)
3.  **Function Registry JSON Schema** (from Builder Lab)
4.  **Adversarial Test Suite** (from Validation Lead Lab)

---

## Part 1: Defining the Brain (Instructions)
Create a new file called `my-agent.yaml`. Start by defining the `instructions`. 

> [!TIP]
> Use the [Master Agent Templates](../templates_and_resources/agents/) as a structural reference for your YAML, JSON, or Rules files.

*   **Action:** Paste the **3-Part Directive** from your context-enriched story into the `instructions` block. 
*   **Why:** This ensures the agent's core logic is identical to the human-validated requirement.

## Part 2: Mounting the Tools (Function Registry)
Reference your Function Registry in the `tools` section of the YAML.
*   **Action:**
    ```yaml
    tools:
      - name: "execute_logic"
        schema: "../03_building_and_coding/function_registry.md"
    ```
*   **Why:** This gives the agent the "Hands" to touch the codebase.

## Part 3: Setting the Guardrails (ADRs & Context)
Tell the agent which architectural rules it MUST follow.
*   **Action:** Add a `context` block pointing to your `/docs/adr/` folder.
*   **Why:** This forces the agent to use RAG to look up your engineering standards before writing code.

## Part 4: The Final Handshake (Activation)
Define the trigger for your agent.
*   **Action:** 
    ```yaml
    on:
      pull_request:
        types: [opened]
    ```

---

## 🏁 Final Submission
Once your `my-agent.yaml` is complete, submit it to the repository. 
**Verification:** Run a local simulation. Ask the agent to "Implement the feature in JIRA-101."
*   Does it find the Story?
*   Does it use the Registry?
*   Does it follow the ADR?

If yes, you have successfully built a **Context Factory Agent**.
